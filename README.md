# scanner_for_extensions

C++17 приложение для Linux/WSL. Оно рекурсивно ищет мультимедийные файлы в заданном каталоге, строит индекс по категориям `audio`, `video`, `images` и периодически обновляет результат.

## Особенности

- **Рекурсивное сканирование**: Полное сканирование каталогов с поддержкой вложенных папок.
- **Категоризация файлов**: Автоматическая классификация по типам:
  - **Audio**: mp3, wav, flac, ogg, aac, m4a
  - **Video**: mp4, avi, mkv, mpg, mpeg, mov, webm
  - **Images**: jpg, jpeg, png, gif, bmp, webp, tiff
- **Периодические обновления**: Инкрементальное сканирование для отслеживания изменений.
- **HTTP сервер**: Встроенный веб-сервер для доступа к индексу через API.
- **JSON вывод**: Структурированный вывод в формате JSON.
- **Опциональная запись файла**: Возможность сохранения индекса в файл `~/.media_files`.

## Требования

- `g++` (компилятор C++)
- `cmake` (система сборки)
- `make` (утилита сборки)

## Сборка

```bash
mkdir -p build
cd build
cmake ..
make
```

## Запуск

### Основное использование

**Сканирование домашнего каталога по умолчанию:**

```bash
./scanner_for_extensions
```

**С указанием каталога и интервала:**

```bash
./scanner_for_extensions --path <PATH_TO_SCAN> --interval 10
```

### HTTP режим

**Запуск HTTP сервера:**

```bash
./scanner_for_extensions --path <PATH_TO_SCAN> --interval 10 --serve
```

**HTTP режим без записи файла на диск:**

```bash
./scanner_for_extensions --path <PATH_TO_SCAN> --interval 10 --serve --no-file
```

### Опции командной строки

| Опция          | Описание                              | По умолчанию |
|----------------|---------------------------------------|-------------|
| `--path DIR`   | Корневой каталог для сканирования     | `$HOME`     |
| `--interval SEC` | Интервал сканирования в секундах    | `60`        |
| `--serve`      | Запустить HTTP сервер на `/media_files` |             |
| `--port PORT`  | Порт HTTP сервера                     | `1234`      |
| `--no-file`    | Не записывать `~/.media_files`, только JSON |         |

## API

JSON индекс доступен по адресу:

**http://localhost:1234/media_files**

**Пример запроса:**

```bash
curl http://localhost:1234/media_files
```

**Формат JSON:**

```json
{
  "audio": ["file1.mp3", "file2.wav"],
  "video": ["file3.mp4"],
  "images": ["file4.jpg", "file5.png"]
}
```

Каждая категория содержит массив строк соответствующими файлами.
